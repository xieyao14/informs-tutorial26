# Generative DRO Reproduction

Two standalone examples:

- `examples/outage`: flow-based scenario generation for county outage counts.
- `examples/portfolio`: flow-generated return scenarios followed by DRO portfolio optimization.

## Setup

```bash
cd generative_dro_repro
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

CUDA is optional but recommended for full reruns.

## Data

- Outage: PowerOutage.us outage-count records in `examples/outage/data/outage_counts.csv`.
- Portfolio: Yahoo Finance adjusted daily close prices in `examples/portfolio/data/yfinance`.

## Run

```bash
cd examples/outage
python train.py
python plot.py

cd ../portfolio
python train.py
python plot.py
```

Useful options:

```bash
python train.py --device cuda --output-dir results/default
python plot.py --output-dir results/default
```

## Formulation

### Shared Flow Model

Let $x_0 \in \mathbb R^d$ be the scaled data vector used by a given example. Sample
$$
x_1 \sim \mathcal N(0,I), \qquad t \sim \mathrm{Unif}(0,1).
$$
Train a vector field $v_\theta$ by flow matching:
$$
x_t = (1-t)x_0 + t x_1,
$$
$$
\min_\theta \; \mathbb E \left[
\left\|v_\theta(x_t,t) - (x_1-x_0)\right\|_2^2
\right].
$$
Synthetic scenarios are generated by reverse Euler from $x(1)\sim\mathcal N(0,I)$ to $x(0)$.

### Outage Example

Let $c_i \in \mathbb N_0^d$ be the outage-count vector for timestamp $i$, where $d=10$ counties. The flow input is
$$
y_i = \log(1+c_i), \qquad
x_i = (y_i-\mu_y)\oslash \sigma_y,
$$
where $\mu_y,\sigma_y\in\mathbb R^d$ are countywise training statistics and $\oslash$ denotes elementwise division. A generated flow sample $\hat x_i$ is mapped back by
$$
\hat y_i = \mu_y + \sigma_y\odot \hat x_i,
\qquad
\hat c_i = \operatorname{round}\!\left((\exp(\hat y_i)-1)_+\right).
$$
Spatial dependence is modeled because each $x_i$ is one joint county vector, not ten independent county samples. The reported MMD uses log-count vectors. The code uses deterministic subsamples of at most 4096 real and generated samples for this metric. For real samples $a_1,\ldots,a_m$ and generated samples $b_1,\ldots,b_n$, define the RBF kernel
$$
k(a,b)=\exp\!\left(-\frac{\|a-b\|_2^2}{2h}\right),
$$
where the RBF lengthscale is chosen by a real-sample median heuristic:
$$
\ell^2=h,
\qquad
h=\operatorname{median}\left\{\|a_i-a_j\|_2^2:1\le i<j\le m\right\}.
$$
The reported empirical MMD is
$$
\widehat{\operatorname{MMD}}^2
=
\frac{1}{m^2}\sum_{i=1}^m\sum_{j=1}^m k(a_i,a_j)
+
\frac{1}{n^2}\sum_{i=1}^n\sum_{j=1}^n k(b_i,b_j)
-
\frac{2}{mn}\sum_{i=1}^m\sum_{j=1}^n k(a_i,b_j).
$$
This is the biased empirical estimator including diagonal kernel terms.

### Portfolio Example

Let $r_i\in\mathbb R^d$, $i=1,\ldots,n$, be raw daily return vectors for $d=6$ assets, and let $\mu$ be the training-sample mean return vector. The portfolio example uses one global scale
$$
s = \frac{1}{nd}
\sum_{i=1}^{n}\sum_{j=1}^d
\left|R^{\mathrm{train}}_{ij}\right|.
$$
The flow model is trained in the scaled space:
$$
x_i=\frac{r_i-\mu}{s}.
$$
A generated flow sample $\hat x_i$ is mapped back to raw returns as
$$
\hat r_i = \mu + s\hat x_i.
$$
With $w=\mathrm{softmax}(\theta)\in\Delta_d$, the scaled softplus shortfall is
$$
\bar q(w) = \frac{q-\mu^\top w}{s},
$$
$$
\ell(x,w)
=
\frac{1}{\beta}
\log\!\left(1+\exp\left(\beta\left(\bar q(w)-x^\top w\right)\right)\right).
$$
The nominal portfolio solves
$$
\min_\theta \; \frac{1}{n}\sum_{i=1}^n \ell(\hat x_i,w(\theta)).
$$
The robust portfolio solves
$$
\min_\theta \; \max_{z_1,\ldots,z_n}
\frac{1}{n}\sum_{i=1}^n
\left[
\ell(z_i,w(\theta))
-
\frac{1}{2\gamma}\|z_i-\hat x_i\|_2^2
\right].
$$
Here $\hat x_i$ and $z_i$ are both in the global-scaled return space; worst-case raw returns are recovered as
$$
r_i^{\mathrm{wc}}=\mu+s z_i.
$$
Evaluation uses raw test returns $r_i^{\mathrm{test}}$ and stress returns
$$
r_i^{\mathrm{stress}} = r_i^{\mathrm{test}}-\delta\mathbf 1.
$$
Portfolio flow metrics use the same empirical MMD formula above, with $a_i$ and $b_j$ in the global-scaled return space.

## Experiment Settings

Outage:

- Data: 10 Atlanta-area counties; 15-minute outage-count vectors.
- Window: `2018-01-01T00:00:00` to `2024-12-31T23:45:00`.
- Scaling: `log1p` counts followed by countywise mean/std scaling.
- Flow model: MLP width 256, depth 4, time embedding 64.
- Flow training: 100 epochs, batch size 1024, AdamW lr `1e-3`, weight decay `1e-5`.
- Sampling: reverse Euler, 100 steps, 4096 nominal samples.
- Metrics: RBF MMD² in `log1p` space; marginal histograms and county-correlation heatmaps.

Portfolio:

- Data: six ETFs, `SPY IWM EFA EEM AGG GLD`; daily adjusted close prices.
- Window: `2023-01-01` to `2025-12-31`; train through `2024-12-31`.
- Scaling: global-scaled returns, $(R-\mu)/s$, with $s$ the mean absolute training return.
- Flow model: MLP width 256, depth 4, time embedding 64.
- Flow training: 20000 epochs, batch size 1024, AdamW lr `1e-3`, weight decay `1e-5`.
- Sampling: reverse Euler, 100 steps, one nominal return sample per training day.
- DRO: softplus shortfall with `q=0.02`, `beta=4`; gammas `0.01 0.1 1 10`.
- DRO training: equal-weight robust initialization; 100000 nominal steps and 100000 robust GDA steps.
- Metrics: RBF MMD² in global-scaled return space; test and stress-test portfolio losses.
